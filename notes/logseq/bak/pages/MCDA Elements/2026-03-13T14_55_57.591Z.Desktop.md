- Chapter 2.2 – Elements of MCDA
  
  ---
- ## 2.2.1 Decision Makers
- An entity (individual, group, or organization) responsible for making decisions
- Distinction between **individual** vs. **group** decision making is based on consistency of goals/preferences/beliefs — not just number of people
- **Interest groups**: three types — proponents, those affected, and those with mediation authority
- **Decision-making agents**: autonomous software programs with properties like reactivity, rationality, and preferences; used in geospatial modeling (geospatial agents)
  
  ---
- ## 2.2.2 Criteria
- Used to evaluate alternatives; includes both **attributes** and **objectives**
- Each criterion must be: comprehensive, measurable, complete, operational, decomposable, non-redundant, and minimal
- Criteria can be **spatially explicit** (e.g., site shape, contiguity) or **spatially implicit** (e.g., distance, slope)
- ### Objectives vs. Attributes
- **Objective**: desired direction of improvement (maximize/minimize)
- **Attribute**: measurable property that operationalizes an objective
- ### Hierarchical Structure
- ![image.png](../assets/image_1773408585202_0.png)
- Goal → Objectives → Sub-objectives → Attributes → Alternatives
- Criterion hierarchies can also include interest groups or scenarios
- Implemented in tools like CommonGIS and ILWIS
  
  ---
- ## 2.2.3 Decision Alternatives
- Courses of action with at least two components: **what** (action) and **where** (location)
- Can be **explicitly spatial** (e.g., site locations, land use patterns) or **implicitly spatial** (spatial implications without explicit location)
- Defined differently in raster vs. vector GIS models
- Variables can be **binary**, **discrete**, or **continuous**
- ### Feasible Alternatives
- ![image.png](../assets/image_1773408602619_0.png)
-
- Constraints split alternatives into feasible vs. infeasible
- Most common: **Boolean constraints** (e.g., must be outside wetlands, must be 1 km from any river)
- ### Non-dominated (Pareto-efficient) Alternatives
- ![image.png](../assets/image_1773408627865_0.png)
- An alternative is non-dominated if no other alternative is at least as good on all criteria and better on at least one
- The best solution should be found within the set of **non-dominated feasible** alternatives
  
  ---
- ## 2.2.4 Decision Matrix
- ![image.png](../assets/image_1773408671208_0.png)
- Organizes all MCDA elements in a table: rows = alternatives (with coordinates), columns = criteria
- Element $a_{ik}$ = performance of alternative $A_i$ on criterion $C_k$
- Criterion weights $w_k$ assigned to each criterion (spatially homogeneous by default; can vary spatially in explicit MCDA)
- For group decisions: multiple decision matrices, one per agent/decision maker

- # Chapter 2.3 – Basic Concepts
  
  
  The three core building blocks of any GIS-MCDA procedure are:
  1. **Value Scaling** – transforming raw criterion values to a common scale
  2. **Criterion Weighting** – expressing the relative importance of criteria
  3. **Combination Rules** – integrating scaled values and weights into an overall ranking
  
  ---
- ## 2.3.1 Value Scaling
  
  Criteria are measured in different units (e.g., km, €, %) and must be **standardized to a common 0–1 scale** before they can be combined. The value 0 = least desirable outcome; 1 = most desirable.
  
  ---
- ### Global Value Function
  
  The general form transforms raw criterion values into standardized scores $v(a_{ik})$ in [0,1]:
- **For criteria to be minimized:** $v(a_{ik}) = ((max - a_{ik}) / r_k)^p$
- **For criteria to be maximized:** $v(a_{ik}) = ((a_{ik}- min) / r_k)^p$
  
  Where rk = max - min is the global range of criterion k, and p > 0 is a shape parameter.
  
  **Shape of the function depending on p:**
- ![image.png](../assets/image_1773410918318_0.png)
- | Value of p | Shape | Risk attitude | Interpretation |
  |---|---|---|---|
  | p = 1 | Linear (score range) | Risk-neutral | Equal marginal utility across all levels |
  | 0 < p < 1 | Concave | Risk-averse | Small gains at low values are valued more |
  | p > 1 | Convex | Risk-seeking | Gains at high values are valued more |
  
  The **linear form** (p = 1) — also called the *score range procedure* — is by far the most commonly used in GIS-MCDA.
  
  ---
- ### Specific Scaling Methods
- #### 1. Score Range (Linear) Standardization
  Rescale raw values to [0, 1] proportionally using the global minimum and maximum. Simple, intuitive, and easy to implement in GIS via map algebra.
- #### 2. Piecewise Linear Approximation (Mid-Point Value Method)
  Used when experts define the value function directly:
- Set 0 and 1 at the minimum and maximum criterion values
- Ask the expert to identify the **mid-value point** (assigned 0.5)
- Repeat recursively to define 0.25, 0.75, 0.125, etc.
- Connect points to form a piecewise linear curve
- Used in tools like MATS and IDRISI
- #### 3. Value Function for Categorical Data (Pairwise Comparison)
  When a criterion is categorical (e.g., land cover type), raw values cannot be rescaled numerically. Instead:
- Conduct pairwise comparisons between all categories
- Derive scores from the principal eigenvector of the comparison matrix (AHP method)
- Example scores: spruce-fir = 1.00, mixed conifer = 0.67, ponderosa pine = 0.08, other = 0.00
- #### 4. Fuzzy Membership Functions
  Implemented in IDRISI's FUZZY module. Each alternative receives a value based on its **degree of membership** in a fuzzy set. Available function shapes:
- **Sigmoidal** – S-shaped transition
- **J-shaped** – rapid rise or fall
- **Linear** – monotonic ramp
- **User-defined** – custom breakpoints
  
  ---
- ### Local Value Function
  
  The **global** value function applies the same transformation everywhere, assuming spatially homogeneous preferences. The **local** form standardizes within a spatial **neighbourhood** q, using the **local range** $r_k^q = max_q - min_q$:
- **For maximization:** $v(a_{ik}^q) = ((a_{ik}^q - min_q) / r_k^q)^p(q)$
- Results in **spatially varying standardization**: a score of 80 may be "best" in one neighbourhood but only "average" in another
  
  **Two ways to define neighbourhood q:**
  
  1. **Discrete zones** – predefined regions (watersheds, urban zones, raster blocks)
  2. **Moving windows** – focal location + surrounding neighbours, defined by:
	- Rook's contiguity (shared edge)
	- Queen's contiguity (shared edge or corner)
	- Distance threshold: all locations within distance d
	- k-nearest neighbours: the p closest locations

- ## 2.3.2 Criterion Weighting
  
  A **weight** $w_k$ reflects the relative importance of criterion k. Weights must:
- Satisfy 0 ≤ $w_k$ ≤ 1 and sum to 1
- Be **ratio-scaled**: if C1 is twice as important as C2, then $w_1 = 2w_2$
- Reflect the **range of criterion values** (*range-sensitive principle*): wider range → higher weight, all else equal
  
  ---
- ### Global Weighting Methods
  
  Assume **spatially homogeneous preferences**: one weight per criterion, applied everywhere.
- #### 1. Ranking (Rank Sum Weights)
- Rank criteria from most to least important (p = 1 = most important)
- Weight formula: $w_k = \frac{(n - p_k + 1)}{\sum_k^{n}(n - p_k + 1)}$, with $p_k$ rankposition of criteria $k$
- **Equal weights** $(w_k = 1/n)$ is a special case — often used but has no theoretical justification
- ✓ Simple and fast
- ✗ No theoretical foundation; breaks down with many criteria
- #### 2. Rating
- Assign a score to each criterion on a fixed scale (e.g., 0–100); most important = 100
- Normalize by dividing each score by the total sum
- ✓ Intuitive and slightly more nuanced than ranking
- ✗ Same theoretical limitations as ranking
- #### 3. Pairwise Comparison (AHP)
  
  The most widely used criterion weighting method in GIS-MCDA (Malczewski 2006).
  Developed by Saaty (1980) as part of the Analytic Hierarchy Process (AHP).
  
  ---
  
  **Step 1 – Compare all pairs of criteria using Saaty's 1–9 scale:**
  
  | Value | Interpretation |
  |---|---|
  | 1 | Equally important |
  | 3 | Moderately more important |
  | 5 | Strongly more important |
  | 7 | Very strongly more important |
  | 9 | Extremely more important |
  | 2, 4, 6, 8 | Intermediate values |
  
  For $n$ criteria, only $\frac{n(n-1)}{2}$ comparisons are needed because the matrix
  is reciprocal: if $C_1$ is rated 3 times more important than $C_2$, then
  $C_2$ is automatically $\frac{1}{3}$ as important as $C_1$.
  
  ---
  
  **Step 2 – Build the pairwise comparison matrix $C$:**
  
  For three criteria $C_1$, $C_2$, $C_3$, suppose:
- $C_1$ is moderately more important than $C_2$ → $c_{12} = 3$
- $C_1$ is strongly more important than $C_3$ → $c_{13} = 5$
- $C_2$ is slightly more important than $C_3$ → $c_{23} = 2$
  
  The full reciprocal matrix is:
  
  $$C = \begin{pmatrix} 1 & 3 & 5 \\ \frac{1}{3} & 1 & 2 \\ \frac{1}{5} & \frac{1}{2} & 1 \end{pmatrix}$$
  
  The diagonal is always 1 (a criterion compared to itself).
  Off-diagonal entries satisfy $c_{pk} = \frac{1}{c_{kp}}$.
  
  ---
  
  **Step 3 – Normalize the matrix (column normalization):**
  
  Divide each entry by the sum of its column:
  
  $$\tilde{c}_{kp} = \frac{c_{kp}}{\sum_{k=1}^{n} c_{kp}}$$
  
  $$\tilde{C} = \begin{pmatrix} 0.652 & 0.667 & 0.625 \\ 0.217 & 0.222 & 0.250 \\ 0.130 & 0.111 & 0.125 \end{pmatrix}$$
  
  ---
  
  **Step 4 – Compute weights by averaging each row:**
  
  $$w_k = \frac{1}{n} \sum_{p=1}^{n} \tilde{c}_{kp}$$
  
  $$w_1 = \frac{0.652 + 0.667 + 0.625}{3} = 0.648$$
  
  $$w_2 = \frac{0.217 + 0.222 + 0.250}{3} = 0.230$$
  
  $$w_3 = \frac{0.130 + 0.111 + 0.125}{3} = 0.122$$
  
  Check: $w_1 + w_2 + w_3 = 0.648 + 0.230 + 0.122 = 1.000$ ✓
  
  $C_1$ receives the highest weight (0.648), reflecting that it was judged
  most important in the pairwise comparisons.
  
  ---
  
  **Step 5 – Check consistency (Consistency Ratio CR):**
  
  Human judgment is rarely perfectly consistent. For example, if:
- $C_1$ is 3× more important than $C_2$
- $C_2$ is 2× more important than $C_3$
  
  Then for perfect transitivity, $C_1$ should be $3 \times 2 = 6$× more important
  than $C_3$. If instead $c_{13} = 5$ was entered, this is an inconsistency.
  The method quantifies and tolerates a degree of inconsistency.
  
  Compute the **largest eigenvalue** $\lambda_{max}$:
  
  $$\lambda_{max} \approx \frac{1}{n} \sum_{k=1}^{n} \frac{(Cw)_k}{w_k}$$
  
  For the example above, $\lambda_{max} \approx 3.004$.
  
  Compute the **Consistency Index (CI)**:
  
  $$CI = \frac{\lambda_{max} - n}{n - 1} = \frac{3.004 - 3}{3 - 1} = 0.002$$
  
  Compute the **Consistency Ratio (CR)**:
  
  $$CR = \frac{CI}{RI(n)}$$
  
  Where $RI$ is the average consistency index of a randomly generated matrix:
  
  | $n$ | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
  |---|---|---|---|---|---|---|---|
  | $RI$ | 0.00 | 0.52 | 0.89 | 1.11 | 1.25 | 1.35 | 1.40 |
  
  For the example: $CR = \frac{0.002}{0.52} = 0.004$
- **CR < 0.10**: acceptable level of consistency → proceed with the weights
- **CR ≥ 0.10**: judgments are too inconsistent → revise the pairwise comparisons
  
  ---
  
  **Transitivity and the meaning of weights:**
  
  The principle of transitivity requires that if $C_1 \succ C_2$ and $C_2 \succ C_3$,
  then $C_1 \succ C_3$. More precisely, if $C_1$ is 3× more important than $C_2$
  and $C_2$ is 2× more important than $C_3$, perfect consistency demands:
  
  $$c_{13} = c_{12} \times c_{23} = 3 \times 2 = 6$$
  
  The CR measures how far the actual judgments deviate from this ideal.
  Since $\lambda_{max} = n$ for a perfectly consistent matrix, any excess
  above $n$ signals inconsistency.
  
  ---
  
  **Important note on weights and criterion ranges:**
  
  Criterion weights cannot be interpreted in isolation from the **ranges of criterion values**.
  If $C_1$ has a much wider range than $C_2$, $C_1$ will dominate the result even with
  equal weights. The pairwise comparison method does not explicitly account for this —
  it asks only for relative importance, not for importance conditional on the value range.
  This is considered a key limitation of the method (Goodwin & Wright 1998).
  
  ✓ Most widely used; theoretically grounded; intuitive elicitation; handles inconsistency  
  ✗ Questions are ambiguous — relative importance is asked without reference to criterion ranges  
  ✗ Can be burdensome with many criteria: $n = 10$ requires 45 comparisons

- #### 4. Entropy-Based Weights
  Does **not** require decision-maker input — derived from information content of the data.
  
  **Procedure:**
  1. Compute proportional values: $p_{ik} = a_{ik} / \sum_i(a_{ik})$
  2. Compute entropy: $E_k = -\sum_i(p_{ik} × ln(p_{ik})) / \ln(m)$
  3. Compute diversity: $b_k = 1 - E_k$
  4. Compute weights: $w_k^E = b_k / \sum_k(b_k)$
  
  **Logic:**
- More diverse criterion values → lower entropy → higher weight
- A perfectly homogeneous criterion gets weight = 0 (can be dropped)
- Can be combined with other weights: $w_k^E* = (w_k^E × w_k) / \sum_k(w_k^E × w_k)$
  
  ✓ Objective; no elicitation needed
  ✗ Rarely used in practice; strict underlying requirements
  
  ---
- ### Spatially Explicit Weighting Methods
  
  Account for **spatial heterogeneity of preferences**: weight of a criterion varies by location.
- #### 5. Proximity-Adjusted Weights
  Weight of criterion k for alternative i depends on distance to a reference location j:
  1. Standardize distances: dij^s = min{dij} / dij (closer = higher value)
  2. Compute weight: wik = wk × (dij^s / mean(dij^s))
- Alternatives **closer** to the reference location receive **higher** criterion weights
- Makes spatial bias toward a location explicit
- Total weight is redistributed, not inflated
- #### 6. Range-Based Local Weights
  Based on the range-sensitive principle — within each neighbourhood, weight is proportional to local range:
  
  $w_k^q = (wk × rk^q / rk) / sum_k(wk × rk^q / rk)
- Criteria that vary more locally receive higher local weights
- Used in Local WLC and Local OWA models
- #### 7. Entropy-Based Local Weights
  Entropy computed within each neighbourhood q:
- $Ek^q = -\sum_{i in q}(pik^q × ln(pik^q)) / ln(|q|)$
- $wEk^q = (1 - Ek^q) / \sum_k(1 - Ek^q)$
  
  Same logic as global entropy but applied locally.

- ## 2.3.3 Combination Rules
  
  A **combination rule** integrates criterion maps and weights into an overall ranking of alternatives.
  
  ---
- ### Compensatory vs. Non-Compensatory
  
  **Compensatory** – low value on one criterion can be offset by a high value on another (trade-offs allowed):
- Example: Weighted Linear Combination (WLC)
  
  **Non-compensatory** – no trade-offs; each criterion must independently pass a threshold:
- **Conjunctive (AND)**: alternative must meet *all* thresholds → strict screening
- **Disjunctive (OR)**: alternative must meet *at least one* threshold → permissive
- **Lexicographic**: rank by most important criterion first; use subsequent criteria only to break ties
  
  ---
- ### Multiattribute (MADA) vs. Multiobjective (MODA)
  
  | Feature | MADA | MODA |
  |---|---|---|
  | Alternatives | Explicitly defined, finite set | Implicitly defined via decision variables |
  | Criteria | Attributes (measurable quantities) | Objectives (maximize/minimize functions) |
  | Constraints | Implicit | Explicit |
  | Decision paradigm | Evaluation / choice | Design / search / optimization |
  | Spatial examples | Site selection, suitability mapping | Location-allocation, routing, districting |
  | Methods | WLC, AHP, ELECTRE, TOPSIS | Goal programming, compromise programming, heuristics |
  
  ---
- ### Discrete vs. Continuous
- **Discrete**: finite set of alternatives (typical of MADA)
	- Each alternative = point in decision space (map) and criterion space (scatterplot)
	- Non-dominated (Pareto-efficient) solutions identifiable in criterion space
- **Continuous**: infinite feasible region defined by constraints (typical of MODA)
	- Feasible region = bounded area in decision space
	- Non-dominated frontier = Pareto front (a curve) in objective space
	- Solved via mathematical programming or heuristics
	  
	  **Site selection vs. site search:**
- **Site selection** (MADA/discrete): candidate sites known; rank and evaluate them
- **Site search** (MODA/continuous): no predefined candidates; find optimal site shape, boundary, and location simultaneously
  
  ---
- ### Spatially Implicit vs. Spatially Explicit MCDA
  
  Four types based on whether criteria and alternatives are spatially explicit or implicit (van Herwijnen & Rietveld):
  
  | Type | Alternatives | Criteria | Example |
  |---|---|---|---|
  | 1 | Spatially explicit | Spatially explicit | Site selection with shape/compactness criteria |
  | 2 | Spatially explicit | Spatially implicit | Siting analysis using proximity/accessibility |
  | 3 | Spatially implicit | Spatially explicit | Policy alternatives with spatial side-effects |
  | 4 | Spatially implicit | Spatially implicit | Aspatial MCDA applied to spatial data |
  
  **Two aggregation paths (von Herwijnen 1999):**
- **Path 1 – Spatial aggregation first, then MCDA:**
  Summarize each alternative to one value per criterion spatially → apply MCDA → one ranked output map
- **Path 2 – MCDA first, then spatial aggregation:**
  Apply MCDA location-by-location → spatially aggregate results
  
  The two paths **can yield different rankings** — the choice of path is analytically significant.
